## 0. 준비 — 어떤 툴로 찍나

### 방법 A) DBeaver / pgAdmin (권장 — 블록 실행이 편함)
1. 연결: Host `localhost` / Port `5432` / DB `skala_db` / User `skala_user` / PW `0000`
2. SQL 에디터에 `sql/02_tuning_Q01-Q11.sql` 내용을 붙여넣기.
3. 문항별로 **쿼리 블록을 드래그 선택 → 실행**(DBeaver: `Ctrl+Enter`).
4. 화면에 **① 쿼리문 + ② 실행계획(EXPLAIN 결과 그리드)** 이 함께 보이게 캡처.
   - EXPLAIN 결과는 텍스트 그리드로 나오므로 그대로 캡처하면 됨.

### 방법 B) psql 터미널 (한 화면에 쿼리+결과가 깔끔)
```bash
PGPASSWORD=0000 psql -h localhost -U skala_user -d skala_db
```
```
\pset pager off          -- 결과가 페이저로 잘리지 않게
SET search_path = ecom, public;
```
- 문항별 쿼리를 붙여넣고 실행 → **쿼리 + EXPLAIN 결과가 한 화면**에 나옴.
- 터미널 폰트를 키우고(가독성), 창을 넓혀 계획이 줄바꿈으로 뭉개지지 않게.

### macOS 캡처 단축키
- `Cmd + Shift + 4` : 영역 선택 캡처 (권장)
- `Cmd + Shift + 4` → `Space` : 창 전체 캡처
- 저장 위치를 이 `docs/` 폴더로 지정하거나, 찍은 뒤 이동.

> 팁: `now()` 기준 상대시간 데이터라 **오늘 세션에서 바로** 찍는 게 안정적.
> 데이터를 재적재하면 수치가 조금 달라짐.

## 2. Q10 튜닝 전 캡처 요령 (주의)

Q1에서 만든 `idx_orders_revts` 인덱스가 Q10에도 쓰이므로, **Q10 "튜닝 전"** 계획을
보려면 인덱스를 잠깐 지운 뒤 실행해야 한다. `02` 파일 Q10 섹션의 주석을 해제:

```sql
-- Q10 튜닝 전 캡처용
DROP INDEX IF EXISTS idx_orders_revts;
ANALYZE orders;
-- (여기서 Q10 EXPLAIN 실행 → Seq Scan 계획 캡처 = Q10_before.png)

-- 다시 만들어 튜닝 후 캡처
CREATE INDEX idx_orders_revts ON orders(order_ts)
  WHERE order_status IN ('paid','shipped','delivered');
ANALYZE orders;
-- (Q10 EXPLAIN 다시 실행 → Bitmap Index Scan = Q10_after.png)
```

## 3. 캡처 시 꼭 보이게 할 것
- **쿼리문 전체** (무슨 쿼리인지)
- **EXPLAIN (ANALYZE) 실행계획** — 특히 아래 키워드가 보이면 좋다:
  - 튜닝 전: `Seq Scan`, `Rows Removed by Filter`, `SubPlan`
  - 튜닝 후: `Bitmap Index Scan`, `Index Scan`, `WindowAgg`
  - `Execution Time: ... ms` (전/후 비교의 핵심 근거)
- 결과값 쿼리(예: Q1 `net_sales`)는 별도로 한 번 더 실행해 결과 그리드도 캡처하면 완벽.
